README Reflection
Project Summary
This program was created because I wanted a way to track and display frequencies of items in a grocery store input file. Instead of having to count items that are purchased manually, this program can read in a file while storing each item and how many times they occur. From there the user can search through these items through a menu. Options include searching for items, printing all frequencies, and printing a histogram. This allows businesses to visualize customers buying habits.
What I Did Well
I feel that my program was well organized with the use of functions and a class to store everything. Creating a map to store item frequencies allowed my program to run efficiently and stay organized. And last but definitely NOT least, I made sure my menu was straightforward and easy to use so the user would know how to operate the program.
Where I Could Improve
I think I could improve my program by implementing better input validation for unexpected user input. I think my code can always be improved by breaking up my code even more to make my program more scalable. I can always include more comments and documentation.
Challenges and How I Overcame Them
The biggest challenge for me was handling file input and ensuring that the data was being stored and counted properly. Originally I had formatting problems and problems ensuring the file was being read. I eventually solved this problem by testing chunks of my code and looking at class examples/code as well as documentation to better understand how files are used in C++.
Transferable Skills
This project allowed me to become more comfortable working with maps, file I/O, and functions. These topics will be useful in future projects when needing to work with large amounts of data or creating programs that take user input.
Maintainability and Readability
To improve readability and maintainability I structured the program into functions with well defined roles. Making it so that you can change small portions of code without needing to rewrite everything. I also used descriptive names for my variables. And consistently formatted my code.
